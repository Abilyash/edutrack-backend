1. Введение
1.1 Основания для разработки
В условиях цифровизации образования требуется инструмент, который не просто хранит файлы, но и анализирует процесс обучения. "EduTrack" создается как высоконадежная платформа, ориентированная на автоматизацию проверки работ и мониторинг успеваемости.
1.2 Назначение разработки
Цель — создание LMS-платформы (Learning Management System) с микросервисной перспективой, где студенты могут сдавать работы, а преподаватели — управлять курсами и получать аналитику рисков с использованием современных стандартов безопасности (GDPR, шифрование).
2. Анализ целевой аудитории
2.1 Целевая аудитория
Студенты: Технические специалисты, сдающие лабораторные работы (Java, Go, C++).
Преподаватели: Инструкторы, нуждающиеся в автоматизации учета оценок и дедлайнов.
Администраторы: Сотрудники, управляющие доступом и аудитом системы.

3.2 Нефункциональные требования
Безопасность: Шифрование персональных данных в Supabase (PGSQL) на уровне полей.
Производительность: Время ответа API < 500мс (кроме тяжелых отчетов).
Масштабируемость: Готовность к развертыванию в Kubernetes (k8s).
3.3 Технические требования
Backend: Java 17+, Spring Boot 3.x.
Database & Auth: Supabase (PostgreSQL).
Messaging: Apache Kafka.
Cache: Redis.
Frontend: React (TypeScript).
4. Архитектура системы
4.1 Общая структура (Clean Architecture)
Проект реализуется согласно Hexagonal Architecture:
Domain: Чистые бизнес-объекты и интерфейсы.
Application (Service): Сценарии использования (Use Cases), маппинг DTO (MapStruct).
Infrastructure: Реализация репозиториев (Spring Data JPA), интеграция с Supabase API и Kafka Producers.
Web (Controller): REST эндпоинты с валидацией входных данных.
4.2 Модули системы
Core Module: Базовый функционал обучения.
Auth Module: Интеграция с Supabase GoTrue.
Analytics Module: Обработка данных из Kafka для построения графиков успеваемости.

5. Проектирование интерфейса
5.1 UX/UI дизайн
Dashboard Студента: Прогресс-бар по курсу, ближайшие дедлайны.
Панель Преподавателя: Список студентов с цветовой индикацией риска (зеленый/желтый/красный).
6. Реализация и тестирование
6.1 Этапы разработки
Sprint 1: Настройка Supabase, Auth, базовая архитектура Spring Boot.
Sprint 2: Модуль курсов и загрузки файлов в Storage.
Sprint 3: Интеграция Kafka и создание сервиса аудита.
Sprint 4: Тестирование (Testcontainers) и настройка CI/CD.
6.2 Методология тестирования
Unit: Покрытие логики мапперов и сервисов (JUnit 5).
Integration: Использование Testcontainers для имитации реальной среды (Postgres, Kafka, Redis).
Security: Тестирование JWT-фильтров и доступа к методам (Spring Security Test).
Load: Тестирование через k6/Gatling (сценарий: 100 одновременных загрузок лаб).
7. DevOps и развертывание
7.1 CI/CD Pipeline (GitHub Actions)
Build: Компиляция Maven.
Test: Запуск модульных и интеграционных тестов.
Docker: Сборка образа и пуш в GitHub Container Registry.
Deploy: Обновление манифестов в k3s/minikube.



8. Заключение
Реализация "EduTrack" на стеке Java/Spring/Supabase обеспечит создание профессионального продукта, соответствующего стандартам индустрии 2026 года. Платформа станет фундаментом для внедрения ИИ-помощников в образовательный процесс.
